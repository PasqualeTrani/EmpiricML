## How to Contribute

### Reporting Bugs

1. Check if the bug has already been reported in [GitHub Issues](https://github.com/PasqualeTrani/EmpiricML/issues)
2. If not, create a new issue with:
   - A clear title and description
   - Steps to reproduce the bug
   - Expected vs actual behavior
   - Your environment details

### Suggesting Features

Open an issue with the `enhancement` label describing your feature request.

### Pull Requests

1. Fork the repository
2. Create a feature branch: `git checkout -b feature/your-feature`
3. Make your changes
4. Add tests if applicable
5. Submit a pull request

## Development Setup

```bash
git clone https://github.com/PasqualeTrani/EmpiricML.git
cd EmpiricML
pip install -e ".[dev]"
```

## Code Style

- Follow PEP 8 guidelines
- Add docstrings to all public functions
- Write unit tests for new features
